# EduPulse 판단 기준 총정리

모든 페이지에서 표시되는 등급, 점수, 권장값이 어떤 기준으로 산출되는지 정리한 문서.

---

## 1. 수요 등급 (Demand Tier)

모든 페이지의 기본이 되는 핵심 분류. 모델이 예측한 **과정 단위 수강 신청 인원수** 하나로 결정된다.
(모델은 주간 단위로 학습, `predict_demand()`에서 `ENROLLMENT_SCALE(×8)` 적용 후 분류)

| 등급 | 영문 | 조건 | 색상 |
|---|---|---|---|
| 높음 | High | 예측 인원 **40명 이상** | 초록 (success) |
| 보통 | Mid | 예측 인원 **20명 이상 ~ 40명 미만** | 노란 (warning) |
| 낮음 | Low | 예측 인원 **20명 미만** | 빨간 (error) |

- 코드: `edupulse/constants.py` → `DEMAND_THRESHOLDS`, `classify_demand()`, `ENROLLMENT_SCALE`
- 임계값: `{"high": 40, "mid": 20}` (과정 단위)

---

## 2. 폐강 위험도 (Closure Risk)

대시보드에서 표시. 수요 등급을 뒤집어서 폐강 관점으로 재해석한 것. 별도 모델 없음.

**최소 개강 인원: 15명**

| 위험 등급 | 표시 | 조건 | 위험 점수 |
|---|---|---|---|
| high | 위험 (빨간) | 수요 등급 = Low | `1.0 - 예측인원 / 30` |
| medium | 주의 (노란) | 수요 등급 = Mid | 고정 `0.4` |
| low | 안전 (초록) | 수요 등급 = High | 고정 `0.1` |

보조 지표: 신뢰 구간 하한이 15명 미만이면 추가 경고 표시.

| 위험 등급 | 권장 조치 |
|---|---|
| high | 마케팅 강화 + 조기 등록 할인. 개강 4주 전 최소 인원 미달 시 폐강 검토 |
| medium | 조기 등록 혜택 제공. 개강 2주 전 재평가 |
| low | 현재 전략 유지. 개강 준비 집중 |

- 코드: `edupulse/api/routers/demand.py` → `assess_closure_risk()`

---

## 3. 마케팅 타이밍 (Marketing Timing)

마케팅 페이지에서 표시. 수요 등급에 따라 **고정 규칙표**로 결정.

| 수요 등급 | 광고 시작 시점 | 얼리버드 기간 | 권장 할인율 |
|---|---|---|---|
| High (높음) | 개강 **2주 전** | 7일 | 5% |
| Mid (보통) | 개강 **3주 전** | 14일 | 10% |
| Low (낮음) | 개강 **4주 전** | 21일 | 15% |

- 코드: `edupulse/api/routers/marketing.py` → `_MARKETING_RULES`
- 로직: 수요 낮을수록 더 일찍, 더 오래, 더 높은 할인

---

## 4. 강사 및 강의실 배정 (Schedule)

운영 페이지에서 표시. 예측 수강생 수 기반 단순 나눗셈.

| 항목 | 계산 공식 | 상수 |
|---|---|---|
| 필요 강사 수 | `ceil(예측 수강생 / 15)` | 강사 1명당 15명 |
| 필요 강의실 수 | `ceil(예측 수강생 / 30)` | 강의실당 최대 30명 |

- 코드: `edupulse/api/routers/schedule.py`
- 반 편성: 강의실 수만큼 A반, B반, ... 자동 생성

---

## 5. 잠재 수강생 전환 예측 (Lead Conversion)

마케팅 페이지에서 표시. 상담 로그 기반.

### 예상 전환 수강생 수
```
예상 전환 = 최근 8주 평균 상담 건수 x 기본 전환율(30%)
```

### 수요 추세 판단 (current_demand_tier)
최근 8주 상담 건수를 전반부 4주 vs 후반부 4주로 비교:

| 조건 | 결과 |
|---|---|
| 후반부 > 전반부 x 1.05 | High (증가) |
| 후반부 < 전반부 x 0.95 | Low (감소) |
| 그 외 | Mid (안정) |

### 상담 데이터 없을 때
웹 로그의 `page_views / 100`을 대리 지표로 사용.

- 코드: `edupulse/api/services/marketing_service.py`
- 기본 전환율: `DEFAULT_CONVERSION_RATE = 0.3` (30%, 고정값)

## 6. 시뮬레이터 시나리오

시뮬레이터 페이지에서 표시. 기준 예측값에 단순 배율 적용.

| 시나리오 | 계산 | 예시 (기준=32명) |
|---|---|---|
| Baseline (기준) | 모델 예측값 그대로 | 32명 |
| Optimistic (낙관) | 예측값 x **1.2** | 38명 |
| Pessimistic (비관) | 예측값 x **0.8** | 26명 |

각 시나리오별로 수요 등급 자동 계산.

### 예상 매출 (수강료 입력 시)
수강료는 **선택 입력**이며, 수요 예측 자체에는 영향을 주지 않는다. 입력 시 보조 정보로 매출을 추정한다.

```
예상 매출 = 예측 수강생 수 × 수강료
```

| 시나리오 | 예시 (기준=32명, 수강료=100만원) |
|---|---|
| Baseline | 32명 × 100만원 = **3,200만원** |
| Optimistic | 38명 × 100만원 = **3,800만원** |
| Pessimistic | 26명 × 100만원 = **2,600만원** |

수강료 미입력 시 매출 항목은 표시되지 않고 수강생 수만 표시.

- 코드: `edupulse/api/services/simulation_service.py` → `simulate_new_course()`

## 7. 최적 개강일 추천 점수 (Composite Score)

시장 분석 페이지에서 표시. 탐색 기간 내 모든 월요일 후보에 대해 산출.

### 점수 구성 (0~100점)

| 요소 | 가중치 | 데이터 소스 | 의미 |
|---|---|---|---|
| 예측 수강생 수 | 30% | 앙상블 모델 예측 | 후보 중 최대값 대비 정규화 (0~1) |
| 채용 공고 점수 | 20% | `job_postings.csv` | 최근 채용 건수 / 분야 평균 (최대 1.0) |
| 경쟁 여유도 | 20% | `competitor_data.csv` | `1 - (경쟁사 개강 수 / 평균의 2배)` |
| 계절성 검색 점수 | 30% | `search_trends.csv` | 해당 ISO 주차의 과거 연도 평균 검색량 / 전체 평균의 2배 |

### 점수 스케일링
원시 점수를 전체 후보군 내에서 **55~92점 범위로 재스케일링**하여 분별력 확보.
- 최고 후보 → 92점 근처
- 최저 후보 → 55점 근처
- 후보가 1개이면 → 74점 (중간값)

### 부가 정보 (점수에 미반영, 참고용)
| 항목 | 데이터 소스 | 판단 기준 |
|---|---|---|
| 검색 트렌드 | `search_trends.csv` | 최근 4주 vs 이전 4주 평균: >5% 상승, <5% 하락, 그 외 안정 |
| 경쟁사 수 | `competitor_data.csv` | 해당 날짜 직전 최근 `competitor_openings` 값 |

- 코드: `edupulse/api/services/simulation_service.py` → `find_optimal_start_dates()`

## 8. 시장 포화도 (Saturation Index)

시장 분석 페이지 > 경쟁 학원 동향에서 표시.

### 산출
```
포화도 지수 = 현재 경쟁사 개강 수 / 전체 기간 평균 개강 수  (최대 2.0)
```

### 등급

| 포화도 지수 | 등급 | 표시 |
|---|---|---|
| 1.5 이상 | high | 포화 (빨간) |
| 0.8 이상 ~ 1.5 미만 | medium | 보통 (노란) |
| 0.8 미만 | low | 여유 (초록) |

### 포화도별 경쟁 대응 조언

| 포화도 지수 | 조언 |
|---|---|
| 1.5 이상 | 차별화된 커리큘럼 또는 틈새 분야 집중 권장 |
| 1.0 이상 | 가격 경쟁력 또는 강사 전문성 강화 권장 |
| 0.5 이상 | 마케팅 강화로 시장 점유율 확대 가능 |
| 0.5 미만 | 블루오션 — 선점 전략으로 적극 개강 권장 |

- 코드: `edupulse/api/services/simulation_service.py` → `get_competitor_analysis()`

## 9. 수강생 인구통계 트렌드

시장 분석 페이지 > 수강생 인구통계에서 표시.

### 트렌드 뱃지
`student_profiles.csv`를 날짜순 정렬 후 전반부 vs 후반부 수강생 합계 비교:

| 조건 | 표시 |
|---|---|
| 후반부 > 전반부 x 1.05 | 증가 (초록) |
| 후반부 < 전반부 x 0.95 | 감소 (빨간) |
| 그 외 | 안정 (파란) |

### 연령대 분포
`age_20s_ratio`, `age_30s_ratio`, `age_40plus_ratio` 컬럼의 분야별 평균값.

### 수강 목적 분포
`purpose_career`, `purpose_hobby`, `purpose_cert` 컬럼의 분야별 평균값.
→ 취업/이직, 취미/교양, 자격증으로 표시.

- 코드: `edupulse/api/services/simulation_service.py` → `get_demographics_breakdown()`
