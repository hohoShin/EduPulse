# AI 활용 리포트

**프로젝트명** : EduPulse — AI 기반 수강 신청 수요 예측 솔루션  
**공모전 주제** : AI 활용 차세대 교육 솔루션  
**작성일** : 2025년 4월 7일

---

## 1. 개요

본 리포트는 EduPulse 프로젝트 기획 및 설계 과정에서 AI(Claude)를 어떻게 활용하였는지를 기록한 문서입니다.
아이디어 발굴부터 데이터 파이프라인 설계, 프로젝트 문서화에 이르기까지 전 과정에 걸쳐 AI와의 대화를 통해 의사결정을 진행하였으며, 그 흐름과 결과물을 정리합니다.

---

## 2. AI 활용 단계별 요약

### 2-1. 아이디어 발굴

**활용 방식** : AI에게 공모전 주제(AI 활용 차세대 교육 솔루션)와 주최 기관(코딩·보안·게임·아트 학원)의 특성을 제공하고, AI 분석 및 예측 모델 기반의 솔루션 아이디어를 요청하였습니다.

**AI가 제안한 주요 아이디어**

| 번호 | 아이디어 | 대상 |
|---|---|---|
| 1 | 학습 이탈 예측 & 선제적 개입 시스템 | 수강생 |
| 2 | 개인별 학습 속도 예측 & 커리큘럼 자동 재편성 | 수강생 |
| 3 | 포트폴리오 품질 예측 & 피드백 자동화 | 수강생 |
| 4 | 강사 수업 품질 분석 시스템 | 강사 |
| 5 | 번아웃 예측 시스템 | 수강생 |
| 6 | 적성 진단 & 전공 트랙 추천 | 수강생 |
| 7 | 수강 신청 수요 예측 모델 | 운영자 |
| 8 | 강사-학생 매칭 예측 | 강사·수강생 |

**선택 결과** : 교육 운영자의 실질적 페인포인트를 해결하고, 비즈니스 임팩트가 명확한 **수강 신청 수요 예측 모델**을 핵심 방향으로 선정하였습니다.

---

### 2-2. 서비스 범위 구체화

**활용 방식** : 수요 예측 모델과 연계 가능한 서비스 아이디어를 AI에게 추가 요청하여 솔루션의 범위를 확장하였습니다.

**AI가 제안한 연계 서비스**

운영 효율화 방향으로는 강사 수급 자동 스케줄링, 최적 개강 시점 추천, 폐강 리스크 예측을 제안하였습니다. 마케팅·매출 연계 방향으로는 동적 수강료 최적화, 광고 타이밍 예측, 잠재 수강생 전환 예측을 도출하였으며, 전략 기획 방향으로는 신규 강좌 개설 시뮬레이터, 지역·연령별 수요 세분화, 경쟁 학원 동향 연동 예측을 제안하였습니다.

**결과** : 수요 예측이라는 단일 모델을 운영·마케팅·전략 세 방향으로 연결하는 구조를 확정하였습니다.

---

### 2-3. 필요 데이터 정의

**활용 방식** : 수요 예측 모델 구현에 필요한 데이터 목록을 AI와 함께 도출하고, 수집 난이도 및 우선순위를 정리하였습니다.

**도출된 데이터 분류**

내부 데이터로는 수강 이력 DB, 상담 로그, 학생 프로필, 웹/앱 로그를 정의하였습니다. 외부 데이터로는 네이버 데이터랩·Google Trends 검색 트렌드, 채용 공고 크롤링, 자격증 시험 일정, 경쟁 학원 모니터링 데이터를 정의하였습니다.

**AI의 추가 인사이트** : 실제 데이터 없이도 데이터 수집·전처리·모델링 흐름을 설계하는 것 자체가 공모전 평가 대상임을 제언하여 파이프라인 설계 문서화로 방향을 잡았습니다.

---

### 2-4. 데이터 파이프라인 설계

**활용 방식** : AI에게 파이프라인 설계 지원을 요청하였고, 인터랙티브 시각화 다이어그램을 생성하였습니다.

**설계된 파이프라인 4단계**

수집 단계에서는 내부·외부 데이터를 수집하고, 전처리 단계에서는 결측치·이상치 정제, 시계열 변환, 멀티소스 병합을 수행합니다. 모델링 단계에서는 Prophet·LSTM·XGBoost 세 모델을 학습·검증하며, 서비스 출력 단계에서는 운영 대시보드, 마케팅 알림, 전략 리포트를 제공합니다.

**산출물** : 단계별 펼치기/접기가 가능한 인터랙티브 파이프라인 다이어그램(React 컴포넌트)

---

### 2-5. 프로젝트 문서화

**활용 방식** : 대화 전체 맥락을 기반으로 AI가 프로젝트 문서를 직접 생성하고, 피드백에 따라 반복 수정하였습니다.

**생성된 문서 목록**

| 파일 | 내용 | 수정 횟수 |
|---|---|---|
| `README.md` | 프로젝트 개요, 기능, 파이프라인, 기술 스택, 파일 구조 | 5회 |
| `CLAUDE.md` | 코딩 에이전트용 컨텍스트 및 개발 지침 (영문) | 4회 |
| `requirements.txt` | 필요 라이브러리 목록 | 1회 |

**문서 수정 이력 요약**

README.md는 프로젝트명 EduPulse 반영, 분야별 특화 아이디어 및 연계 솔루션 섹션 제거, 실행 환경 섹션 추가, S3 → scp 전환 순으로 수정되었습니다. CLAUDE.md는 M4 MPS 설정 추가, 로컬/서버 분리 전략으로 환경 섹션 전면 개편, S3 내용 제거, 전체 영문 재작성 순으로 수정되었습니다.

---

### 2-6. 기술 환경 의사결정

**활용 방식** : 실제 개발 환경(M4 맥북 + Digital Ocean Droplet)을 공유하고, AI로부터 환경별 최적 전략을 도출하였습니다.

**주요 의사결정 내용**

모델 학습 환경 분리 결정으로, Droplet의 1 vCPU / 2GB RAM 사양으로는 LSTM 재학습이 어렵다는 점을 AI가 분석하여, LSTM은 맥북에서 수동 학습 후 scp로 전송하고 XGBoost·Prophet만 서버에서 자동 재학습하는 방식을 채택하였습니다.

Apple Silicon 최적화 설정으로, MPS 디바이스 자동 감지 코드, Prophet의 conda 설치 권장 이유, LSTM 배치 사이즈 권장값 등 M4 환경 특화 내용을 CLAUDE.md에 반영하였습니다.

S3 미사용 결정으로, 초기 설계에 포함되었던 AWS S3를 실제 프로젝트 규모와 맞지 않는다고 판단하여 제거하고, 모델 파일 전송은 scp 방식으로 간소화하였습니다.

## 5. 결론

본 프로젝트는 기획 단계 전반에 걸쳐 AI를 단순한 정보 검색 도구가 아닌 **공동 기획자**로 활용하였습니다.
아이디어 발굴, 범위 설정, 데이터 정의, 파이프라인 설계, 문서 작성, 기술 환경 결정에 이르기까지 AI와의 반복적인 대화를 통해 의사결정의 속도와 품질을 높일 수 있었습니다.
특히 실제 개발 환경의 제약 조건(서버 사양, Apple Silicon 이슈 등)을 AI가 분석하고 현실적인 대안을 제시한 점은 실무적 관점에서 높은 활용 가치를 보여주었습니다.
