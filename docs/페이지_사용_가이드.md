# EduPulse 페이지 사용 가이드

각 페이지의 목적, 조작 방법, 화면 구성 요소를 정리한 문서.
판단 기준의 상세 수치는 [판단_기준_정리.md](판단_기준_정리.md)를 참고.

## 1. 대시보드 (`/`)

### 목적
운영 전체 현황을 한 화면에서 파악하는 커맨드 센터.
"지금 뭘 먼저 해야 하나?"를 빠르게 답해주는 페이지.

### 조작
- **분야 선택**: 우측 상단 셀렉터로 코딩/보안/게임/아트 전환 → 모든 지표가 해당 분야로 갱신
- **새로고침**: 우측 상단 버튼 클릭 시 전체 데이터 재조회
- **데모 모드** (개발 환경만): 성공/로딩/빈값/오류 상태를 수동 전환하여 UI 테스트 가능

### 화면 구성

| 영역 | 내용 | 읽는 법 |
|---|---|---|
| **현재 수요 신호** | 수요 등급 뱃지 (높음/보통/낮음) | 현재 분야의 AI 예측 결과. 높음=초록, 보통=노란, 낮음=빨간 |
| **긴급 운영 리스크** | 가장 중요한 경고 1건 | 폐강 위험, 수요 급변 등 즉시 대응 필요한 알림 |
| **다음 액션** | 권장 행동 안내 | 경고 상태에 따라 마케팅/운영 대응 제안 |
| **운영 기준선** | 보조 지표 카드들 | 예상 수강생 수, 전환율, 상담 건수 등 기본 운영 수치 |
| **예상 수요 트렌드** | 꺾은선 차트 (최근 30일) | 실제 수요 흐름 + 예측 범위 (신뢰 구간) |
| **시스템 알림** | 전체 경고 목록 | 심각도별(critical/warning/info) 정렬된 전체 알림 |

### 활용 흐름
```
분야 선택 → 수요 신호 확인 → 리스크 있으면 운영/마케팅 페이지로 이동 → 없으면 현 상태 유지
```

## 2. 시뮬레이터 (`/simulator`)

### 목적
신규 강좌 개설 전에 "이 강좌를 열면 몇 명이나 올까?"를 미리 예측.
기준/낙관/비관 3가지 시나리오를 비교하고, 실제 강좌 조건을 입력해 맞춤 브리프를 생성.

### 조작
1. 상단 **시나리오 비교 카드** 3장은 자동 표시 (입력 불필요)
2. 좌측 폼에 **강좌명 + 분야 + 개강 예정일** 입력 (수강료는 선택)
3. **시뮬레이션 실행** 클릭
4. 우측에 맞춤 브리프 생성
5. **조건 변경** 버튼으로 다른 조건 재시도 가능

### 입력 필드

| 필드 | 필수 | 설명 |
|---|---|---|
| 강좌명 | O | 자유 텍스트 (예: "고급 게임 엔진 디자인") |
| 분야 | O | 코딩/보안/게임 개발/아트 & 디자인 중 택 1 |
| 개강 예정일 | O | 미래 날짜만 가능 |
| 수강료 | X | 원 단위 입력. 수요 예측에는 영향 없음. 입력 시 시나리오별 예상 매출(`수강생 수 × 수강료`) 계산에만 사용 |

### 화면 구성

| 영역 | 내용 | 읽는 법 |
|---|---|---|
| **시나리오 비교 카드** | 기준/낙관/비관 3장 | 과거 데이터 기반 참고 수치. 기준선에 상단 강조 표시 |
| **핵심 예측 지표** | 예상 수요 인원 + 신뢰 구간 | AI 모델이 예측한 수강생 수와 오차 범위 |
| **AI 모델 신뢰도** | 사용된 모델명 | ensemble/xgboost/prophet 중 어떤 모델로 예측했는지 |
| **운영 리소스** | 필요 강사 수 + 강의실 수 | 예측 인원 기반 자동 계산 |
| **마케팅 액션** | 광고 시작 시점 + 할인율 | 수요 등급에 따른 권장 마케팅 전략 |
| **낮은 수요 경고** | Low 등급일 때만 표시 | 폐강 위험 안내 + 대응 방안 제안 |

### 활용 흐름
```
시나리오 카드로 시장 감 파악 → 실제 조건 입력 → 브리프 확인 → 운영/마케팅 페이지로 이동
```

## 3. 마케팅 분석 (`/marketing`)

### 목적
잠재 수강생의 전환 상황을 파악하고, 수요 등급별 최적 광고 타이밍을 결정.

### 조작
- **분야 선택**: 상단 셀렉터로 전환 → 전환 예측 + 타이밍 추천이 해당 분야로 갱신
- **새로고침**: 우측 상단 버튼으로 데이터 재조회

### 화면 구성

| 영역 | 내용 | 읽는 법 |
|---|---|---|
| **이번 달 예상 전환 수** | 숫자 + 전월 대비 변동률 | 최근 8주 상담 건수 평균 x 전환율(30%)로 산출. +%=증가, -%=감소 |
| **최근 상담 트렌드** | 막대 차트 (8주) | 주간 상담 건수 추이. 우상향이면 관심 증가 신호 |
| **추천 액션** | 텍스트 + 바로가기 링크 | 상담 추세(증가/감소/유지)에 따른 마케팅 권고. 분야별 특화 팁 포함 |
| **광고 타이밍 추천** | 등급별 3장 카드 | High/Mid/Low 각각의 광고 시작 시점, 얼리버드 기간, 할인율. 현재 수요에 해당하는 카드가 강조 표시 |

### 광고 타이밍 카드 읽는 법
- **강조 표시된 카드** = 현재 분야의 수요 등급에 해당하는 전략
- 나머지 카드는 참고용 (수요가 변하면 전략도 바뀜)
- "현재 수요" 뱃지가 붙은 카드의 수치를 따르면 됨

### 활용 흐름
```
분야 선택 → 전환 수/추세 확인 → 현재 수요에 맞는 타이밍 카드 확인 → 광고 집행 계획 수립
```

## 4. 운영 관리 (`/operations`)

### 목적
특정 강좌의 폐강 위험도를 평가하고, 강사/강의실 배정 계획을 수립.

### 조작
1. 좌측 폼에 **강좌명 + 분야 + 개강 예정일** 입력
2. **분석 실행** 클릭
3. 우측에 분석 결과 브리프 생성

### 입력 필드

| 필드 | 필수 | 설명 |
|---|---|---|
| 강좌명 | O | 분석 대상 강좌 이름 |
| 분야 | O | 코딩/보안/게임 개발/아트 & 디자인 |
| 개강 예정일 | O | 분석 기준 날짜 |

### 화면 구성

| 영역 | 내용 | 읽는 법 |
|---|---|---|
| **상단 브리프 헤더** | ACTION REQUIRED / ON TRACK | 빨간 배경 = 폐강 위험 높음 (즉시 조치), 초록 뱃지 = 정상 개강 가능 |
| **상황 요약** | 예측 수강생 vs 최소 개강 인원 | 미달 시 몇 명 부족한지 + 권장 조치 안내 |
| **바로가기 버튼** | 마케팅/시뮬레이터 링크 | 위험 높을 때만 표시. 마케팅 강화 또는 일정 재조정으로 연결 |
| **위험도 게이지** | 퍼센트 바 + 위험/주의/안전 | 점수가 높을수록 위험. 게이지 색상으로 직관적 판단 |
| **위험도 추이** | 미니 꺾은선 차트 | 최근 8주간 위험도 변화 추세 |
| **주요 위험 요인** | 텍스트 목록 | 왜 위험한지 구체적 원인 (예: "신뢰 구간 하한이 최소 인원 미만") |
| **강사/강의실 배정** | 숫자 + 배정 테이블 | 필요 인력과 공간. 시간대 드롭다운으로 변경 가능 |

### 활용 흐름
```
강좌 정보 입력 → 분석 실행 → ACTION REQUIRED면 마케팅/시뮬레이터 이동
                            → ON TRACK이면 배정 테이블 확인 후 확정
```

## 5. 시장 분석 (`/market`)

### 목적
신규 강좌 개설의 전략적 의사결정을 위한 3단계 분석 보드.
타겟 파악 → 경쟁 확인 → 최적 일정 도출.

### 조작
- **분야 선택**: 상단 셀렉터로 전환
- **탐색 시작일/종료일**: 최적 개강일 추천 범위 설정 (최대 16주)
- **데이터 갱신**: 우측 상단 버튼으로 전체 재조회

### 화면 구성 (3단계)

#### Step 1. 타겟 분석 — 수강생 인구통계

| 요소 | 내용 | 읽는 법 |
|---|---|---|
| **트렌드 뱃지** | 증가/안정/감소 | 최근 수강생 수 추세. 증가=초록, 안정=파란, 감소=빨간 |
| **총 수강생 수** | 숫자 | 해당 분야 누적 수강생 |
| **연령대 분포** | 파이 차트 | 20대/30대/40대 이상 비율. 마우스 오버 시 추정 인원수 표시 |
| **수강 목적 분포** | 가로 막대 차트 | 취업·이직/취미·교양/자격증 비율 |

#### Step 2. 경쟁 환경 — 경쟁 학원 동향

| 요소 | 내용 | 읽는 법 |
|---|---|---|
| **신규 경쟁 학원** | 숫자 + 전분기 대비 증감 | 현재 경쟁사 개강 수. ↑=경쟁 심화, ↓=경쟁 완화 |
| **평균 수강료** | 금액 + 전분기 대비 증감 | 경쟁사 수강료 수준. 가격 전략 참고 |
| **시장 포화도 게이지** | 퍼센트 바 + 포화/보통/여유 | 포화(빨간)=진입 어려움, 여유(초록)=블루오션 |
| **경쟁 대응 조언** | 텍스트 | 포화도에 따른 구체적 전략 제안 |

#### Step 3. 전략적 결론 — 최적 개강일 추천

| 요소 | 내용 | 읽는 법 |
|---|---|---|
| **최우선 추천 일정** | 날짜 + 등급 + 점수 바 | 가장 유리한 개강일. 추천 적합도 점수(0~100점)가 높을수록 좋음 |
| **예상 수강생/경쟁 강도/검색 트렌드** | 숫자와 라벨 | 해당 날짜의 핵심 지표 3가지 |
| **추천 사유** | 텍스트 | 왜 이 날짜가 좋은지 근거 설명 |
| **대안 일정** | 우측 카드 리스트 | 2~5순위 후보. 날짜 + 등급 + 점수 |

### 활용 흐름
```
분야 선택 → Step 1에서 타겟 확인 → Step 2에서 경쟁 확인
→ 탐색 기간 설정 → Step 3에서 최적 개강일 확인 → 시뮬레이터에서 상세 분석
```

## 페이지 간 연결 흐름

```
대시보드 ──── 현황 파악, 리스크 감지
  │
  ├─→ 마케팅 분석 ──── 전환율 확인, 광고 타이밍 결정
  │
  ├─→ 운영 관리 ────── 폐강 위험 평가, 강사/강의실 배정
  │     │
  │     └─→ 시뮬레이터 ── 위험 시 일정 재조정
  │
  └─→ 시장 분석 ────── 경쟁 환경 파악, 최적 개강일 도출
        │
        └─→ 시뮬레이터 ── 추천 일정으로 상세 시뮬레이션
```

### 일반적인 사용 시나리오

**시나리오 A: 일상 모니터링**
> 대시보드 → 수요 신호 확인 → 알림 없으면 종료

**시나리오 B: 신규 강좌 기획**
> 시장 분석 → 타겟/경쟁 파악 → 최적 개강일 확인 → 시뮬레이터에서 조건 입력 → 마케팅 전략 수립

**시나리오 C: 기존 강좌 위험 대응**
> 대시보드에서 경고 감지 → 운영 관리에서 폐강 위험 확인 → 마케팅 분석에서 광고 강화 → 재평가
